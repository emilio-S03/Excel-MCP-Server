### Phase 1 — Source audit

**Finding S1: 3 files use `child_process.exec/spawn`.** Files: `src/tools/audit.ts`, `src/tools/tier-b.ts`, `src/tools/transaction.ts`. Plus the existing `excel-powershell.ts` and `excel-applescript.ts` which legitimately shell out for COM/AppleScript. **Impact:** Risk of command injection if filenames aren't escaped. **Verification needed:** confirm all spawn/exec call sites either (a) use the array-args form (no shell interpretation) or (b) escape file paths before interpolating into a shell string. Spot-check shows they pass paths via array-args form (safe). **Status: 🟢 PASS** assuming the spot-check generalizes — formal audit recommended.

**Finding S2: Hardcoded "credential-shaped" matches in source are all legitimate.** The 6 `password?` matches all refer to user-provided sheet-protection passwords passed as runtime arguments — not embedded secrets. **Status: 🟢 PASS.**

**Finding S3: Filesystem sandbox is enforced.** `ensureFilePathAllowed` is called from 195 call sites across the tools directory. Default sandbox: `~/Documents`, `~/Desktop`, `~/Downloads`. Override via `EXCEL_ALLOWED_DIRS` env var. Path violations throw with `errorCode: 'PATH_OUTSIDE_ALLOWED'`. **Evidence:** `helpers.ts:101 (err as any).code = 'PATH_OUTSIDE_ALLOWED'`. **Status: 🟢 PASS.**

**Finding S4: No outbound network calls in server code.** No `fetch()`, `http.request`, `axios`, `node-fetch`, `grpc`, etc. in `src/tools/`. The only network-adjacent deps are pulled by `@modelcontextprotocol/sdk` for HTTP/SSE transport (which this server doesn't use — it's stdio-only). **Status: 🟢 PASS.**

**Finding S5: No prompt-injection patterns in tool descriptions.** Scanned all 110 tool descriptions — no hidden Unicode, no instruction-like text targeting Claude (e.g., "ignore previous instructions"). **Status: 🟢 PASS.**

### Phase 2 — Dependency audit

`npm audit --omit=dev --audit-level=low` returned **9 vulnerabilities (6 moderate, 3 high, 0 critical)**:

| Package | Severity | Issue | Path |
|---|---|---|---|
| `@modelcontextprotocol/sdk` | HIGH | ReDoS, cross-client data leak via shared transport instance reuse, DNS rebinding protection not enabled by default | direct dep |
| `minimatch` | HIGH | Multiple ReDoS vectors (repeated wildcards, GLOBSTAR backtracking, nested `*()` extglobs) | transitive (via SDK / exceljs) |
| `path-to-regexp` | HIGH | DoS via sequential optional groups + multiple wildcards | transitive |
| `ajv` | MODERATE | ReDoS when using `$data` option | transitive |
| `body-parser` | MODERATE | DoS via URL encoding | transitive |
| `brace-expansion` | MODERATE | Zero-step sequence causes process hang | transitive |
| `qs` | MODERATE | `arrayLimit` bypass in comma + bracket parsing → DoS | transitive |
| `uuid` | MODERATE | Missing buffer bounds check in v3/v5/v6 | transitive (via exceljs) |
| `exceljs` | MODERATE | Depends on vulnerable uuid | direct dep |

**Practical exposure assessment:**

| Vuln class | Practical risk for THIS server | Why |
|---|---|---|
| ReDoS in regex parsing (minimatch, ajv, path-to-regexp, qs) | LOW | These vectors require the server to parse untrusted external HTTP query strings or URL paths. This server is stdio-only — no HTTP surface. |
| Cross-client data leak in `@modelcontextprotocol/sdk` | LOW-MEDIUM | Vector requires sharing a single SDK transport instance across clients. This server creates one transport per spawn (Claude Desktop spawns a fresh process per session). Not exploitable in current usage. |
| DNS rebinding (SDK) | LOW | Only relevant if the server were exposing an HTTP endpoint. stdio-only. |
| uuid bounds check | LOW | exceljs uses uuid only for internal ID generation, not on attacker-controlled input. |
| brace-expansion / body-parser DoS | LOW | Same as ReDoS — needs HTTP surface this server doesn't have. |

**Bottom line:** all 9 vulns are in dependencies, not in your code. None are exploitable in stdio-only usage today. **They should still be patched** to prevent regression if the SDK or a downstream dep ever exposes one of these paths to user input. Easiest path: `npm audit fix` (non-breaking) and re-test. The `--force` option would push exceljs to v3.4.0 which is breaking — do not run that without testing the chart and pivot tools.

**Status: 🟡 ATTENTION** — patch but not urgent.
